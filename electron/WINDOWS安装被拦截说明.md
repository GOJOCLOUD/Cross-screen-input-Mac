# Windows 提示「智能应用控制」/ 无法安装 — 说明

这不是安装包「写坏了」，而是 **Windows 11/10 的安全机制**在拦截**未做代码签名**（或尚未积累「声誉」）的安装程序。  
你看到的 **「智能应用控制」**（或类似「已保护你的电脑」「无法验证发布者」）都属于这类情况。

## 为什么会出现

| 现象 | 常见原因 |
|------|----------|
| 直接禁止安装、无法继续 | **智能应用控制** 处于「开启」且判定本安装包不可信 |
| 蓝色/黄色提示窗 | **SmartScreen**（基于声誉，新发布的版本更容易被拦） |
| 仅提示、可点「更多信息」继续 | 同上，用户仍可手动放行 |

**开源 / 小团队**若未购买 **Authenticode 代码签名证书**对 `Setup.exe`、主程序签名，就容易被拦；**与本次版本改了哪些业务代码无必然关系**——只要换了构建机、版本号或刚发布，声誉会重新计算，也可能「以前能装、这次被拦」。

## 用户侧可怎么做（任选其一）

1. **仍要安装（推荐先试）**  
   - 在拦截窗口点 **「更多信息」** → **「仍要运行」** / **「仍要安装」**。  
   - 若安装程序已下载到本地：**右键** → **属性** → 若有 **「解除锁定」**，勾选后确定，再运行。

2. **智能应用控制（Windows 11）**  
   - **设置** → **隐私和安全性** → **Windows 安全中心** → **应用和浏览器控制** → **智能应用控制**。  
   - 可改为 **「评估」**（仅记录，少拦截）或 **「关闭」**（需管理员，且会降低系统防护，仅建议临时使用）。  
   - 具体菜单名称可能随系统语言/版本略有差异。

3. **改试便携版**  
   - 若仓库提供 **portable**（绿色单文件 `KPSR跨屏输入 x.x.x.exe`），有时不会被 NSIS 安装包同一策略拦截，可对比尝试。

4. **企业环境**  
   - 若公司策略禁止未签名软件，需由 IT 加白名单或提供签名版安装包。

## 开发者侧长期方案（治本）

- 购买 **标准代码签名证书**（或 EV），在 CI 里对 **安装包 + 主程序** 签名；积累声誉后 SmartScreen 提示会明显减少。  
- 将安装包提交微软 **Defender** 误报申诉（若被报毒名而非仅声誉拦截）。

---

**结论**：当前「禁止安装 + 提到智能控制」是 **系统安全策略 + 未签名/新发布声誉** 的典型表现，不是业务代码里某一行「导致不能装」。若需要，可在发版说明里附上本文链接，减少用户误解。
